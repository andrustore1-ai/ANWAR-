نسخة R7 Burger المصححة
======================

المطلوب الذي تم تنفيذه:
- لا يوجد مجلدات داخل الحزمة.
- كل صفحة HTML مستقلة.
- CSS مدموج داخل كل ملف HTML.
- جافاسكربت الخاص بكل صفحة مدموج داخل نفس ملف HTML.
- Firebase فقط في ملف مستقل اسمه firebase.js.
- تم حذف Firebase Auth نهائياً، لذلك تم حل خطأ:
  auth/admin-restricted-operation
- المفتاح الافتراضي 0000 يعمل مباشرة من login.html حتى لو Firestore غير مهيأ.

الملفات:
- index.html: منيو الزبون فقط بدون إدارة.
- login.html: تسجيل دخول الأدمن.
- admin.html: إدارة الإعدادات والأقسام والأصناف والمفاتيح.
- tables.html: إضافة الطاولات وطباعة رابط/QR مختلف لكل طاولة.
- kitchen.html: شاشة المطبخ مع عداد إشعارات الطلبات الجديدة.
- reports.html: تقارير مبيعات وفواتير حرارية وطلبات واتساب.
- firebase-check.html: فحص اتصال Firebase وتهيئة البيانات.
- firebase.js: إعداد Firebase ودواله فقط.
- firestore.rules.txt: قواعد Firestore للتجربة.

طريقة التشغيل:
1) ارفع كل الملفات في نفس المكان بدون مجلدات.
2) افتح Firebase Console > Firestore Database > Rules.
3) انسخ محتوى firestore.rules.txt واعمل Publish.
4) افتح firebase-check.html وتأكد أن القراءة والكتابة تعمل.
5) افتح login.html واكتب المفتاح:
   0000
6) من admin.html اضغط تهيئة Firebase، ثم أنشئ مفتاح إدارة جديد من تبويب مفاتيح الدخول.

ملاحظة مهمة:
قواعد firestore.rules.txt مفتوحة للتجربة حتى يعمل المشروع بدون Firebase Auth.
بعد ما يشتغل النظام على الاستضافة النهائية، يجب تأمين القواعد بطريقة أقوى قبل الاستخدام التجاري الحقيقي.
